## 一、前端需求

### P0 - 核心功能（必须完成）

#### 1.1 附件下载功能
**问题描述**: 任务详情页面的附件功能无法下载文档

**解决方案**:
- 检查附件下载 API 端点
- 实现前端下载逻辑（Blob 处理）
- 添加下载进度提示

**验收标准**: 用户能够正常上传和下载任务附件

---

#### 1.2 数据持久化系统
**问题描述**: 任务详情页面的状态变更、历史记录、评论、附件等数据未持久化，刷新页面后数据丢失

**解决方案**:
- **方案 A**: LocalStorage/IndexedDB（前端持久化，适合单机演示）
- **方案 B**: RESTful API + 数据库（推荐，支持多用户协作）

**技术选型**:
| 方案 | 适用场景 | 优点 | 缺点 |
|-----|---------|------|------|
| LocalStorage | 单机演示 | 实现简单，无需后端 | 容量有限，不支持协作 |
| IndexedDB | 大数据量单机 | 容量大，支持索引 | API 复杂 |
| SQLite | 轻量级后端 | 单文件，易部署 | 并发性能一般 |
| PostgreSQL | 生产环境 | 功能强大，稳定 | 需要独立部署 |

**推荐方案**: 采用分层架构
1. 前端: LocalStorage 作为缓存层
2. 后端: FastAPI + PostgreSQL/SQLite 持久化存储
3. API: RESTful 接口实现 CRUD 操作

---

#### 1.3 状态变更确认机制
**问题描述**: 状态变更操作缺少确认步骤，容易误操作

**解决方案**:
- 实现确认弹窗组件
- 显示变更前后的状态对比
- 支持 ESC 键取消操作

**交互设计**:
```
点击 "进行中" 按钮
    ↓
弹出确认框:
┌─────────────────────────────┐
│  确认变更任务状态？          │
│                             │
│  当前状态: 待处理            │
│  变更状态: 进行中            │
│                             │
│  [取消]  [确认变更]          │
└─────────────────────────────┘
```

---

#### 2.3 多模型 API 支持
**需求描述**: 支持多种大模型 API 接入

**支持的模型服务商**:
| 服务商 | API 类型 | 特点 |
|-------|---------|------|
| 阿里云 CodePlan | OpenAI 兼容 | 代码生成优化 |
| 智谱 AI | OpenAI 兼容 | 国产大模型 |
| OpenAI | OpenAI | GPT 系列 |
| 本地 LLM | Ollama/vLLM | 私有化部署 |

**配置方案**:
```json
// config.json
{
  "llm": {
    "providers": [
      {
        "name": "zhipu",
        "type": "openai-compatible",
        "baseUrl": "https://open.bigmodel.cn/api/paas/v4",
        "models": ["glm-4", "glm-4-flash"]
      },
      {
        "name": "local",
        "type": "ollama",
        "baseUrl": "http://localhost:11434",
        "models": ["llama3", "qwen2"]
      }
    ],
    "default": "zhipu/glm-4"
  }
}
```

**前端配置界面**:
- 模型服务商管理
- API Key 配置
- 模型参数调整
- 连接测试

---
