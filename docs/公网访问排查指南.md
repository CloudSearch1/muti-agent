# 公网访问问题排查指南

## 问题现象

Gunicorn 已成功启动，监听 `0.0.0.0:8080`，但无法通过公网 IP + 端口访问。

服务器信息：
- 本机 IP: `172.19.20.66`
- 主机名: `iZ0xi70vyy13fm7bptjxiaZ` (阿里云实例)
- Gunicorn 监听: `0.0.0.0:8080`

---

## 排查步骤

### 第一步：确认端口监听状态

```bash
# 方法1：使用 ss 命令（推荐）
ss -tlnp | grep 8080

# 方法2：使用 netstat
netstat -tlnp | grep 8080

# 方法3：使用 lsof
lsof -i :8080
```

**预期输出：**
```
LISTEN  0  128  0.0.0.0:8080  0.0.0.0:*  users:(("gunicorn",pid=xxx,fd=x))
```

**关键检查点：**
- 监听地址必须是 `0.0.0.0:8080` 或 `:::8080`
- 如果是 `127.0.0.1:8080`，说明只监听本地，需要修改配置

---

### 第二步：本地回环测试

```bash
# 测试本地回环
curl -I http://127.0.0.1:8080

# 测试内网 IP
curl -I http://172.19.20.66:8080

# 测试公网 IP（需要知道公网 IP）
curl -I http://<公网IP>:8080
```

**如果本地回环正常但公网访问失败，问题在防火墙或安全组。**

---

### 第三步：检查阿里云安全组（最常见问题）

#### 3.1 通过控制台检查

1. 登录 [阿里云控制台](https://ecs.console.aliyun.com/)
2. 找到实例 `iZ0xi70vyy13fm7bptjxiaZ`
3. 点击「安全组」→「配置规则」
4. 检查「入方向」规则

#### 3.2 需要添加的规则

| 协议 | 端口范围 | 授权对象 | 说明 |
|------|----------|----------|------|
| TCP | 8080/8080 | 0.0.0.0/0 | 允许所有 IP 访问 8080 端口 |

#### 3.3 使用阿里云 CLI 检查

```bash
# 安装阿里云 CLI
# pip install aliyun-python-sdk-ecs

# 查看安全组规则（需要配置 AccessKey）
aliyun ecs DescribeSecurityGroupAttribute --SecurityGroupId=<安全组ID>
```

---

### 第四步：检查服务器本地防火墙

#### 4.1 检查 ufw（Ubuntu 默认）

```bash
# 查看状态
sudo ufw status verbose

# 如果是 active，检查规则
sudo ufw status numbered

# 允许 8080 端口
sudo ufw allow 8080/tcp

# 重新加载
sudo ufw reload
```

#### 4.2 检查 firewalld（CentOS/RHEL 默认）

```bash
# 查看状态
sudo systemctl status firewalld

# 查看开放的端口
sudo firewall-cmd --list-ports

# 开放 8080 端口
sudo firewall-cmd --permanent --add-port=8080/tcp

# 重新加载
sudo firewall-cmd --reload
```

#### 4.3 检查 iptables（传统方式）

```bash
# 查看规则
sudo iptables -L -n -v

# 检查 8080 端口
sudo iptables -L -n | grep 8080

# 添加规则（如果需要）
sudo iptables -I INPUT -p tcp --dport 8080 -j ACCEPT

# 保存规则
sudo iptables-save > /etc/iptables/rules.v4
```

---

### 第五步：检查 SELinux（仅 CentOS/RHEL）

```bash
# 查看 SELinux 状态
getenforce

# 如果是 Enforcing，临时关闭测试
sudo setenforce 0

# 永久关闭（需要重启）
sudo sed -i 's/SELINUX=enforcing/SELINUX=disabled/' /etc/selinux/config

# 或者允许端口
sudo semanage port -a -t http_port_t -p tcp 8080
```

---

### 第六步：检查网络连通性

```bash
# 从外部测试端口连通性
# 在本地电脑执行：
telnet <公网IP> 8080

# 或使用 nc
nc -zv <公网IP> 8080

# 或使用 nmap
nmap -p 8080 <公网IP>
```

---

### 第七步：检查进程绑定

```bash
# 查看 gunicorn 进程详情
ps aux | grep gunicorn

# 查看进程绑定的端口
sudo ls -l /proc/$(pgrep -f gunicorn | head -1)/fd

# 或使用
sudo netstat -tlnp | grep gunicorn
```

## 常见问题解决方案

### 问题 1：阿里云安全组未配置

**症状：** 本地访问正常，外网无法访问

**解决方案：**
1. 登录阿里云控制台
2. 进入 ECS 实例详情 → 安全组
3. 添加入方向规则：
   - 协议类型：TCP
   - 端口范围：8080/8080
   - 授权对象：0.0.0.0/0

### 问题 2：防火墙阻止

**症状：** 安全组已配置但仍无法访问

**解决方案：**
```bash
# Ubuntu/Debian
sudo ufw allow 8080/tcp
sudo ufw reload

# CentOS/RHEL
sudo firewall-cmd --permanent --add-port=8080/tcp
sudo firewall-cmd --reload
```

### 问题 3：端口只监听本地

**症状：** `ss -tlnp` 显示 `127.0.0.1:8080`

**解决方案：**
修改启动命令或配置文件，确保绑定 `0.0.0.0:8080`：

```bash
# 方式1：命令行参数
gunicorn -b 0.0.0.0:8080 webui.app:app

# 方式2：配置文件 gunicorn.conf.py
bind = "0.0.0.0:8080"
```

### 问题 4：Nginx 反向代理配置问题

**症状：** 使用 Nginx 反代后无法访问

**解决方案：**
检查 Nginx 配置：
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 一键排查脚本

创建文件 `check_network.sh`：

```bash
#!/bin/bash

echo "========== 公网访问排查 =========="
echo ""

# 获取公网 IP
PUBLIC_IP=$(curl -s ifconfig.me 2>/dev/null || curl -s ip.sb 2>/dev/null)
echo "公网 IP: $PUBLIC_IP"
echo ""

# 检查端口监听
echo "=== 端口监听状态 ==="
ss -tlnp | grep 8080 || echo "⚠️ 8080 端口未监听"
echo ""

# 检查本地访问
echo "=== 本地访问测试 ==="
curl -s -o /dev/null -w "状态码: %{http_code}\n" http://127.0.0.1:8080 --connect-timeout 3 || echo "⚠️ 本地访问失败"
echo ""

# 检查防火墙
echo "=== 防火墙状态 ==="
if command -v ufw &> /dev/null; then
    sudo ufw status | grep 8080 || echo "⚠️ ufw 未允许 8080"
fi
if command -v firewall-cmd &> /dev/null; then
    sudo firewall-cmd --list-ports | grep 8080 || echo "⚠️ firewalld 未开放 8080"
fi
echo ""

# 检查 iptables
echo "=== iptables 规则 ==="
sudo iptables -L INPUT -n | grep 8080 || echo "⚠️ iptables 无 8080 规则"
echo ""

# 检查 SELinux
if command -v getenforce &> /dev/null; then
    echo "=== SELinux 状态 ==="
    getenforce
    echo ""
fi

# 检查 Gunicorn 进程
echo "=== Gunicorn 进程 ==="
pgrep -a gunicorn || echo "⚠️ Gunicorn 未运行"
echo ""

echo "========== 排查完成 =========="
echo ""
echo "如果本地访问正常但公网访问失败："
echo "1. 检查阿里云安全组是否开放 8080 端口"
echo "2. 检查服务器防火墙是否允许 8080 端口"
echo "3. 检查是否有其他安全软件阻止"
```

使用方法：
```bash
chmod +x check_network.sh
./check_network.sh
```

## 解决方案总结

| 问题原因 | 解决方法 |
|----------|----------|
| 阿里云安全组未配置 | 控制台添加 8080 入方向规则 |
| ufw 防火墙阻止 | `sudo ufw allow 8080/tcp` |
| firewalld 阻止 | `sudo firewall-cmd --add-port=8080/tcp` |
| iptables 阻止 | `sudo iptables -I INPUT -p tcp --dport 8080 -j ACCEPT` |
| SELinux 阻止 | `sudo setenforce 0` 或配置端口 |
| 端口绑定错误 | 确保绑定 `0.0.0.0:8080` |
| Nginx 配置错误 | 检查反向代理配置 |
