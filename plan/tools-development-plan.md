# Python Tools 详细分步骤开发计划（严格对齐 OpenClaw Tools）

**版本**: v1.0  
**状态**: Execution Plan  
**关联文档**:
- docs/python-tools-interface-spec.md
- 《Python Tools 设计文档（严格对齐 OpenClaw Tools）》

---

## 1. 计划目标

构建一个与 OpenClaw Tools 设计方式一致的 Python Tools 子系统，覆盖策略裁剪、工具注册、统一执行、长任务模型、Web/Browser 安全护栏、循环检测与可观测能力，达到可上线标准。

---

## 2. 实施原则

- **先契约后实现**：先冻结工具契约与策略优先级。
- **先主链路后扩展**：先实现 `allow/deny/profile/byProvider + registry + executor`。
- **安全前置**：高风险工具默认受限，必须显式授权。
- **全程可测**：每阶段有明确验收标准与回归清单。

---

## 3. 里程碑总览

| 里程碑 | 周期 | 目标 |
|--------|------|------|
| **M0** | 1 周 | 规范冻结与基线搭建 |
| **M1** | 1-2 周 | 策略引擎与工具注册中心 |

## 4. 分阶段执行方案

### 4.1 M0 规范冻结与基线搭建（1 周）

**目标**:
- 冻结工具策略语义、组映射、标准错误模型与统一响应 Envelope。

**任务**:
1. 审核并定稿工具设计文档与接口规范。
2. 明确策略优先级实现顺序：`profile -> byProvider -> allow/deny(deny优先)`。
3. 冻结 `group:*` 映射与保留工具名。
4. 搭建目录、日志规范、测试骨架。

**交付物**:
- 规范冻结版本 `v1.0`
- 初始仓库模块骨架
- 测试模板（单测/集成）

**验收**:
- 架构评审通过
- 无 P0 规范歧义

---

### 4.2 M1 策略引擎与工具注册中心（1-2 周）

**目标**:
- 打通工具可见性裁剪与注册查询能力。

**任务**:
1. 实现 ToolsConfig / AgentToolsConfig 解析。
2. 实现 group 展开与通配匹配（大小写不敏感）。
3. 实现 deny > allow 规则与 provider/model 定向收敛。
4. 实现 `ToolRegistry`（register/get/list/unregister）。
5. 实现策略快照（每请求生成 immutable snapshot）。

**交付物**:
- 策略引擎 MVP
- 工具注册中心 MVP
- 策略计算可视化调试日志

**验收**:
- 给定策略输入，裁剪结果稳定可复现
- `byProvider` 仅收窄、不放大

---

### 4.3 M2 工具执行主链路与 `exec/process`（2 周）

**目标**:
- 实现统一工具执行器与长任务生命周期管理。

**任务**:
1. 实现 `ToolCallRequest/Response` 统一分发。
2. 实现 schema 校验、超时、重试、错误标准化。
3. 实现 exec 同步执行路径。
4. 实现 exec(background=true) + `process(list/poll/log/write/kill/clear/remove)`。
5. 实现 agent 级会话隔离（process session scope）。
6. 接入审计日志（工具名、动作、参数脱敏、耗时）。

**交付物**:
- ToolExecutor MVP
- 进程会话管理模块
- 长任务集成测试

**验收**:
- `exec/process` 全链路可用
- 禁用 process 时 exec 自动降级为同步
- 越权访问 session 被阻断

---

### 4.4 M3 Web/Browser/UI/Session 工具族接入（2 周）

**目标**:
- 接入常用工具族并保持统一协议。

**任务**:
1. 接入 `web_search/web_fetch`（缓存、限长、超时）。
2. 接入 `browser`（snapshot-ref-act 交互模型）。
3. 接入 `canvas`、`nodes`、`message`、`cron`、`gateway`。
4. 接入 `sessions_*` 与 `agents_list`，落实可见性限制。
5. 将工具说明与 API schema 双通道同步暴露给模型。

**交付物**:
- 多工具族可运行版本
- 工具能力矩阵文档

**验收**:
- Web 与 Browser 路径职责清晰不混用
- `browser` act 动作可基于 ref 正常执行
- session/agent 可见性规则生效

---

### 4.5 M4 安全加固与循环检测（1-2 周）

**目标**:
- 满足生产安全与稳定性阈值。

**任务**:
1. 实现 web_fetch SSRF 防护（私网拦截 + 重定向重检）。
2. 增加高风险动作同意门禁（camera/screen/remote run）。
3. 实现循环检测器：
   - genericRepeat
   - knownPollNoProgress
   - pingPong
4. 实现全局熔断阈值与告警事件。
5. 完善敏感日志脱敏策略。

**交付物**:
- 安全与循环检测模块
- 安全测试报告

**验收**:
- 私网抓取阻断率 100%
- 循环检测可触发 warning/critical
- 高风险调用无同意不可执行

---

### 4.6 M5 性能优化、灰度与上线（1 周）

**目标**:
- 完成上线前压测、灰度、回滚预案。

**任务**:
1. 压测关键路径（策略计算、工具调度、长任务轮询）。
2. 优化热点（缓存、并发上限、批量日志）。
3. 建立监控看板与告警。
4. 执行灰度发布：`5% -> 25% -> 50% -> 100%`。
5. 预案演练：降级、熔断、回滚。

**交付物**:
- 压测报告
- 灰度复盘报告
- 上线/回滚 Runbook

**验收**:
- 核心 SLA 达标
- 灰度期间无 P0 事故
- 72 小时稳定运行

## 7. 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 策略规则复杂导致行为偏差 | 策略快照 + 金丝雀用例 + 回放校验 |
| 长任务会话泄漏或膨胀 | 生命周期清理、TTL、会话配额 |
| Web 抓取安全边界被绕过 | DNS/IP 双重校验、重定向逐跳校验 |
| 多工具并发造成资源争抢 | 每工具并发上限 + 全局令牌桶 |
| 工具说明与 schema 不一致 | 发布前自动一致性检查（CI gate） |
